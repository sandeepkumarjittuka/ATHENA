# ATHENA Database Design

**Project:** ATHENA - Artificial Trading Hierarchical Engine for Neural Analysis

**Version:** 1.0

---

# Overview

ATHENA uses PostgreSQL as its primary relational database and Redis for caching and real-time data.

Database goals:

* High performance
* Scalability
* ACID compliance
* Historical storage
* Analytics support

---

# Database Architecture

```
Application

↓

FastAPI

↓

Repository Layer

↓

PostgreSQL

↓

Redis Cache
```

# Database Principles

* Normalize where practical.
* Optimize read performance for market scanning.
* Cache frequently accessed live data in Redis.
* Archive historical data periodically.
* Never delete trade history.
